# Cursor Context Usage (컨텍스트 사용량) 안내

> Cursor 채팅 입력창 옆 **원형 게이지**를 클릭하면 나오는 **Context Usage** 패널을 한국어로 풀어 쓴 문서입니다.  
> (AlgeoMath 기능 문서가 아니라, Cursor IDE 사용 참고용)

---

## 한눈에 보기


| 영어 (UI)                   | 한국어                | 의미                                           |
| ------------------------- | ------------------ | -------------------------------------------- |
| **Context Usage**         | 컨텍스트 사용량           | 지금 이 채팅에서 AI가 “한 번에 기억·참고할 수 있는 정보”가 얼마나 찼는지 |
| **73% Full**              | 73% 찼음             | 전체 한도 대비 사용 비율                               |
| **~187.0K / 256K Tokens** | 약 18.7만 / 25.6만 토큰 | 현재 사용량 / 최대 용량                               |
| **View Report**           | 보고서 보기             | 더 자세한 사용량 리포트                                |
| **Token**                 | 토큰                 | AI가 텍스트를 나누어 세는 단위 (글자·단어와 1:1은 아님)          |


**256K = 약 256,000(25만 6천) 토큰**

---

## 원형 게이지가 의미하는 것

입력창 오른쪽의 **원**은 위 패널과 같은 **컨텍스트 창 채움 정도**입니다.

- 흰색이 늘어남 → 대화·파일·도구 정의 등이 쌓여 **한도에 가까워짐**
- 거의 다 찬 뒤 다시 비는 것처럼 보임 → 보통 **오래된 내용을 요약·압축**해 공간을 비운 경우

글자 수 제한이나 단순 타이머가 아닙니다.

---



## 항목별 해석 (패널 막대)

예시 수치(~187K / 256K, 73% Full) 기준 구성:


| 영어                       | 한국어       | 대략 예시  | 설명                                             |
| ------------------------ | --------- | ------ | ---------------------------------------------- |
| **System prompt**        | 시스템 프롬프트  | 453    | Cursor/에이전트가 기본적으로 따르는 내부 지시                   |
| **Tool definitions**     | 도구 정의     | 9.0K   | 파일 읽기·검색·터미널 등 AI가 호출할 수 있는 도구 명세              |
| **Rules**                | 규칙        | 3.7K   | 프로젝트·사용자 규칙 (예: `.cursor/rules`, SVG 아이콘 규칙 등) |
| **Skills**               | 스킬        | 1.3K   | 연결된 Agent Skill 설명·지침                          |
| **Subagent definitions** | 서브에이전트 정의 | 861    | 탐색·리뷰 등 하위 에이전트 역할 정의                          |
| **Conversation**         | 대화        | 171.7K | **가장 큰 비중** — 지금까지의 채팅, 코드 인용, 도구 결과 등         |


합치면 대략 전체 사용량(~187K)이 됩니다. 대화가 길수록 **Conversation**이 대부분을 차지합니다.

## 자주 하는 오해


| 오해             | 실제                                                 |
| -------------- | -------------------------------------------------- |
| 256K = 256만 토큰 | **약 25만 6천** 토큰                                    |
| 원이 입력 글자 수     | **전체 컨텍스트**(대화+규칙+도구+파일 등)                         |
| 꽉 차면 무조건 고장    | 요약·압축되거나, 오래된 맥락이 약해질 수 있음 → **새 채팅**이 도움이 될 때가 많음 |


---



## 사용 팁

1. **게이지가 높게(70%+)** 올라가면 — 긴 작업을 새 채팅으로 나누거나, 불필요한 대용량 첨부·긴 로그 붙여넣기를 줄입니다.
2. **Rules / Skills** — 프로젝트에 규칙을 많이 넣을수록 고정 사용량이 늘어납니다. 꼭 필요한 것만 유지하는 편이 좋습니다.
3. **Conversation** — 같은 채팅에서 큰 파일·긴 도구 출력을 반복하면 빨리 찹니다.
