## 1. 목표 범위 (무엇을 맞출 것인가)

원본 AlgeoMath와의 비교는 **세 층**으로 나눈다. **이 프로젝트는 2번만 목표**로 한다.


| #     | 비교 기준                             | 로드맵 전부 완료 시(추정)               | 이 프로젝트  |
| ----- | --------------------------------- | ----------------------------- | ------- |
| 1     | 이 레포가 정한 카탈로그·체크리스트               | ~90–95%                       | 참고용     |
| **2** | **원본 작도 화면** (`/app/make` 에디터 UX) | ~55–70% (도구만) → **깊이 보강 시 ↑** | **목표**  |
| 3     | 원본 사이트 전체 (포털·계정·커뮤니티·클라우드)       | ~15–25%                       | **비목표** |




### 2번에 포함

- 기하 작도 · DAG 종속성 · 대수창 연동
- 선택·측정·변환·꾸미기·텍스트·(선택) 그림
- 로컬 JSON 저장·불러오기, 설정(격자·스냅·테마 등)
- 작도 조작감(미리보기·스냅·가이드) — `[ux_guidelines.md](ux_guidelines.md)`



### 2번에 포함하지 않음 (3번)

- Next.js 포털, 로그인/카카오, 커뮤니티·모둠
- 클라우드 저장·공유 플랫폼
- 원본 WASM/CAS·MathQuill **전체** 재현 (부분 강화는 §4 참고)



### 핵심 원칙

> **도구 개수를 100% 채우는 것보다**, 선택·측정·변환·저장 + 스타일/Undo/대수 연동 **깊이가 2번 점수**를 올린다.  
> 블록코딩·사용자 도구·동영상은 2번 필수에서 제외한다.

## 4. 로드맵 밖 보강 (깊이) — 에이전트 구현 힌트

도구 `stub → done`만으로는 2번이 안 오른다. 아래를 **선택(7-1) 직후~병행**으로 본다.

### D1. 스타일/속성 패널 (M5)

- 선택 객체(들)에 대해: stroke/fill 색, 선 굵기, 점선, 라벨 표시, (해당 시) 채우기 투명도
- 대수창 인라인 편집(4-1)과 **역할 분리**: 대수창 = 좌표·식, 패널 = 시각 속성
- 아이콘은 SVG 전용 — `[icon_guidelines.md](icon_guidelines.md)`



### D2. 측정 ↔ 대수 (M2 강화)

- 예: `a = Distance(A,B)` / 각도·넓이 객체가 부모 이동 시 `recompute`
- 대수창 리스트에 측정 행 표시·하이라이트



### D3. Undo 그래프 스냅샷 (M6)

- 현재: 수식 히스토리 중심(4-2)
- 목표: `exportState` / import 기반 스택 또는 커맨드 패턴
- `AlgeoEngine.exportState` 이미 존재 — 저장(12-4)과 **같은 직렬화**를 쓰는 것을 권장



### D4. 선택 UX (S5)

- 클릭 단건 · 드래그 박스 · Shift 토글
- 선택 집합에 대한 이동(자유점만 / 제약 존중)
- Delete / 숨김(H) / 스타일 패널 연동



### D5. 히트·스냅 (S6)

- 작도 중 기존 점·교점·객체 위 흡착 우선순위
- `[ux_guidelines.md](ux_guidelines.md)` 미리보기 원칙 유지

## 5. 권장 작업 순서 (2번 전용)

구현·에이전트는 **아래 순서를 기본**으로 한다. 기존 README 7→11 순차와 다를 수 있음 — **이 문서가 우선**.

```text
1. 7-1 선택 · 그룹선택          (M1)
2. 스타일/속성 패널 + Undo 강화   (M5, M6)  ← 로드맵 단계 번호 없음, 병행
3. 8-1 측정 (+ 대수 연동 D2)     (M2)
4. 9 변환 (타일 제외 가능)        (M3)
5. 12-4 JSON 저장·불러오기       (M4)  — Undo 직렬화와 맞춤
6. 8-2 꾸미기 · 7-2 텍스트       (S1, S2)
7. 그림 넣기 · 11-3 설정          (S4, S3)
8. 터치 · 단축키 · 펜 · 기타 Could
─── 여기까지가 2번 “실무 완성선” ───
✕ 블록코딩 · 사용자 도구 · 동영상 · 포털
```



### 한 줄 요약

**선택 → (스타일·Undo) → 측정 → 변환 → 저장 → 꾸미기/텍스트 → 그림/설정**

## 6. 완료 판정 (2번 “실무 완성선”)

다음을 모두 만족하면 2번 목표의 **실무 완성선**으로 본다.

- [ ] 객체를 선택·다중선택하고 이동·삭제·스타일 변경 가능
- [ ] 길이·각도·넓이 측정이 화면에 표시되고, 부모 이동 시 갱신
- [ ] 대칭·회전·평행이동·확대가 종속 객체로 동작
- [ ] JSON으로 저장 후 새로고침·불러오기로 장면 복원
- [ ] Undo가 작도·이동·삭제를 되돌림 (스타일 포함 권장)
- [ ] 꾸미기 또는 텍스트로 설명 가능한 수업용 장면 구성 가능
- [ ] 블록코딩·포털 없이도 데모/세미나 시연에 무리 없음

원본 대비 정량 추정(깊이 보강 후): 작도 에디터 **대략 70–85%** 체감 (도구 나열만 했을 때의 55–70%보다 상승).
